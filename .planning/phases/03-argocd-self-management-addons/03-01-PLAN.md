<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub OAuth -> ArgoCD Dex | External identity provider; untrusted until validated by Dex |
| OCI Vault -> K8s Secret | OAuth credentials cross from cloud to cluster via ESO |
| User browser -> ArgoCD UI | Unauthenticated until GitHub SSO completes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | ArgoCD Dex SSO | mitigate | GitHub OAuth validates org membership; non-members denied by `policy.default: role:none` |
| T-03-02 | Elevation of Privilege | ArgoCD admin account | mitigate | `admin.enabled: "false"` in configs.cm disables local admin login |
| T-03-03 | Elevation of Privilege | ArgoCD exec | mitigate | `exec.enabled: "false"` in configs.cm disables terminal-in-UI |
| T-03-04 | Elevation of Privilege | Container root access | mitigate | `runAsNonRoot: true`, `readOnlyRootFilesystem: true`, drop ALL caps on all components |
| T-03-05 | Information Disclosure | OAuth credentials in git | mitigate | Credentials in OCI Vault, synced via ESO; values.yaml uses `$client_id` env var substitution |
| T-03-06 | Tampering | ArgoCD self-managed sync loop | mitigate | `ignoreDifferences` on argocd-secret + `RespectIgnoreDifferences=true` |
</threat_model>
