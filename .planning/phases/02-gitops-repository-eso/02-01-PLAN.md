1. Create `bootstrap/control-plane/addons/cluster-addons-appset.yaml` — the single matrix ApplicationSet per D-04:
```yaml
apiVersion: argoproj.io/v1alpha1
kind: ApplicationSet
metadata:
  name: cluster-addons
  namespace: argocd
spec:
  goTemplate: true
  goTemplateOptions: ["missingkey=error"]
  generators:
  - matrix:
      generators:
      # Generator 1: cluster — le o Bridge Secret
      - clusters:
          selector:
            matchLabels:
              argocd.argoproj.io/secret-type: cluster
      # Generator 2: git — descobre diretorios de addons
      - git:
          repoURL: '{{.metadata.annotations.addons_repo_url}}'
          revision: '{{.metadata.annotations.addons_repo_revision}}'
          directories:
          - path: bootstrap/control-plane/addons/*
          # Exclui o proprio appset yaml (nao e um diretorio de addon)
  template:
    metadata:
      name: 'addon-{{.path.basename}}'
    spec:
      project: default
      source:
        repoURL: '{{.metadata.annotations.addons_repo_url}}'
        targetRevision: '{{.metadata.annotations.addons_repo_revision}}'
        path: 'bootstrap/control-plane/addons/{{.path.basename}}'
      destination:
        namespace: '{{.path.basename}}'
        name: '{{.name}}'
      syncPolicy:
        automated:
          prune: true
          selfHeal: true
        syncOptions:
        - CreateNamespace=true
        - ServerSideApply=true
  syncPolicy:
    preserveResourcesOnDeletion: true
```

NOTE on feature flag gating (D-04 + D-05 tension documented in RESEARCH.md): The matrix generator creates Applications for ALL discovered addon directories regardless of `enable_*` labels. For Phase 2, all labels are `"true"` in the Bridge Secret so this is not an issue. Feature flag filtering will be revisited in Phase 3 if needed. The directory presence itself is the gate — only committed addon dirs generate Applications.

|----------|-------------|
| Git repo -> ArgoCD | ArgoCD auto-syncs manifests from gitops-setup — malicious manifest = cluster compromise |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | gitops-setup repo | mitigate | ArgoCD syncs only from `main` branch (targetRevision: main); GitHub branch protection on main required (operator responsibility) |
| T-02-02 | Elevation | ApplicationSet template | accept | ApplicationSet runs in argocd namespace with ArgoCD's service account; it can only create Applications in allowed projects (default project has no cluster resource write) |
| T-02-03 | Information Disclosure | values.yaml files | mitigate | No secrets in git — all credentials via OCI Vault + ESO ExternalSecrets; values files contain only non-sensitive config |
</threat_model>
