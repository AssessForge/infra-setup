1. Create `cluster-secret-store.yaml` per D-07 — ClusterSecretStore with Instance Principal and namespace restriction:
```yaml
apiVersion: external-secrets.io/v1
kind: ClusterSecretStore
metadata:
  name: oci-vault
  annotations:
    argocd.argoproj.io/sync-wave: "2"
spec:
  conditions:
  - namespaceSelector:
      matchLabels:
        kubernetes.io/metadata.name: argocd
  provider:
    oracle:
      vault: "PLACEHOLDER_VAULT_OCID"
      region: "sa-saopaulo-1"
      principalType: InstancePrincipal
```

NOTE: The `vault` field requires the actual OCI Vault OCID. After `terraform apply` in infra-setup, the operator must replace `PLACEHOLDER_VAULT_OCID` with the output from `terraform output vault_ocid`. Add a YAML comment above the placeholder: `# Substituir pelo OCID do Vault: terraform -chdir=../infra-setup/terraform/infra output vault_ocid`. The Bridge Secret annotation `oci_vault_ocid` carries this value but the ClusterSecretStore is a standalone CR not templated by the ApplicationSet.

2. Create `external-secret-github-oauth.yaml` per D-08, ESO-03 — ExternalSecret for ArgoCD Dex GitHub OAuth:
```yaml
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: argocd-dex-github-secret
  namespace: argocd
  annotations:
    argocd.argoproj.io/sync-wave: "2"
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: oci-vault
    kind: ClusterSecretStore
  target:
    name: argocd-dex-github-secret
    creationPolicy: Owner
  data:
  - secretKey: client_id
    remoteRef:
      key: github-oauth-client-id
  - secretKey: client_secret
    remoteRef:
      key: github-oauth-client-secret
```

The `remoteRef.key` values MUST exactly match the `secret_name` in `terraform/infra/modules/oci-vault/main.tf`: `github-oauth-client-id` and `github-oauth-client-secret` (Pitfall 5).

3. Create `external-secret-repo-creds.yaml` per D-08, ESO-04 — ExternalSecret for ArgoCD repo credentials:
```yaml
apiVersion: external-secrets.io/v1
kind: ExternalSecret
metadata:
  name: argocd-repo-creds
  namespace: argocd
  annotations:
    argocd.argoproj.io/sync-wave: "2"
spec:
  refreshInterval: 1h
  secretStoreRef:
    name: oci-vault
    kind: ClusterSecretStore
  target:
    name: argocd-repo-creds
    creationPolicy: Owner
    template:
      type: Opaque
      metadata:
        labels:
          argocd.argoproj.io/secret-type: repo-creds
      data:
        type: "Z2l0"
        url: "aHR0cHM6Ly9naXRodWIuY29tL0Fzc2Vzc0Zvcmdl"
        password: "{{ .gitops_repo_pat }}"
        username: "eC10b2tlbg=="
  data:
  - secretKey: gitops_repo_pat
    remoteRef:
      key: gitops-repo-pat
```

IMPORTANT per Pitfall 6: The `target.template.metadata.labels` MUST include `argocd.argoproj.io/secret-type: repo-creds` or ArgoCD won't recognize this as a credential template. The template uses static base64-encoded values for `type` (git), `url` (https://github.com/AssessForge), and `username` (x-token), while `password` is dynamically injected from the ESO data. The `url` value `https://github.com/AssessForge` acts as a credential template matching all repos under the AssessForge org.

NOTE on ESO-05 (notification tokens): Explicitly skipped per D-08 — "if applicable" and no notification system is in v1 scope.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ESO pod -> OCI Vault API | Instance Principal authenticates via node metadata endpoint; secrets cross network boundary |
| OCI Vault -> K8s Secret | Secrets materialized in argocd namespace; any pod in that namespace could read them |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Information Disclosure | ClusterSecretStore | mitigate | `namespaceSelector` restricts secret creation to `argocd` namespace only (D-07); pods in other namespaces cannot request secrets from this store |
| T-02-05 | Spoofing | Instance Principal auth | mitigate | Dynamic Group matching rule scoped to OKE worker node instances only (Phase 1 IAM); IAM policy grants only `read secret-family` — cannot write or delete Vault secrets |
| T-02-06 | Information Disclosure | ExternalSecret repo-creds | mitigate | GitHub PAT stored in OCI Vault (never in git); K8s Secret created only in argocd namespace; PAT should be scoped to read-only access on gitops-setup repo |
| T-02-07 | Tampering | ExternalSecret manifests in git | mitigate | ExternalSecrets only reference Vault secret names — actual secret values never appear in git; changing remoteRef.key to a non-existent name causes sync failure, not data leak |
</threat_model>
