The file must contain three resources in this order:

**Resource 1 — helm_release.argocd (BOOT-01, BOOT-04, BOOT-05):**
```hcl
# --- ArgoCD via Helm (bootstrap minimo) ---

resource "helm_release" "argocd" {
  name             = "argocd"
  repository       = "https://argoproj.github.io/argo-helm"
  chart            = "argo-cd"
  version          = "9.5.0"   # PINNED — app v3.3.6
  namespace        = "argocd"
  create_namespace = true
  wait             = true
  timeout          = 600

  values = [
    yamlencode({
      server = {
        service = { type = "ClusterIP" }
        # TLS terminado no Envoy Gateway (Phase 3)
        extraArgs = ["--insecure"]
      }
      configs = {
        params = {
          # Desabilita exec por seguranca
          "server.exec.enabled" = "false"
        }
      }
    })
  ]

  # OBRIGATORIO: impede Terraform de conflitar com ArgoCD self-management
  lifecycle {
    ignore_changes = all
  }
}
```

**Resource 2 — kubernetes_secret.gitops_bridge (BOOT-02, per D-05, D-06, D-07):**
```hcl
# --- GitOps Bridge Secret ---

resource "kubernetes_secret" "gitops_bridge" {
  metadata {
    name      = "in-cluster"
    namespace = helm_release.argocd.namespace

    labels = {
      # Requerido pelo ArgoCD cluster generator
      "argocd.argoproj.io/secret-type" = "cluster"

      # Metadata
      environment  = "prod"
      cluster_name = var.cluster_name

      # Feature flags para addon ApplicationSet (D-06)
      enable_argocd         = "true"
      enable_eso            = "true"
      enable_envoy_gateway  = "true"
      enable_cert_manager   = "true"
      enable_metrics_server = "true"
    }

    annotations = {
      # Routing do repositorio GitOps (D-07)
      addons_repo_url      = var.gitops_repo_url
      addons_repo_revision = var.gitops_repo_revision

      # Metadata OCI para Helm values dos addons (D-05)
      oci_region            = var.region
      oci_compartment_ocid  = var.compartment_ocid
      oci_vault_ocid        = var.vault_ocid
      oci_public_subnet_id  = var.public_subnet_id
      oci_private_subnet_id = var.private_subnet_id
    }
  }

  # Campos obrigatorios para ArgoCD cluster secret
  data = {
    name   = "in-cluster"
    server = "https://kubernetes.default.svc"
    config = jsonencode({
      tlsClientConfig = { insecure = false }
    })
  }

  depends_on = [helm_release.argocd]
}
```

**Resource 3 — kubectl_manifest.bootstrap_app (BOOT-03):**
```hcl
# --- Root Bootstrap Application ---

resource "kubectl_manifest" "bootstrap_app" {
  yaml_body = yamlencode({
    apiVersion = "argoproj.io/v1alpha1"
    kind       = "Application"
    metadata = {
      name       = "bootstrap"
      namespace  = "argocd"
      finalizers = ["resources-finalizer.argocd.argoproj.io"]
    }
    spec = {
      project = "default"
      source = {
        repoURL        = var.gitops_repo_url
        targetRevision = var.gitops_repo_revision
        path           = "bootstrap/control-plane"
      }
      destination = {
        server    = "https://kubernetes.default.svc"
        namespace = "argocd"
      }
      syncPolicy = {
        automated = {
          prune    = false
          selfHeal = true
        }
        syncOptions = ["CreateNamespace=true", "ServerSideApply=true"]
      }
    }
  })

  depends_on = [
    helm_release.argocd,
    kubernetes_secret.gitops_bridge,
  ]
}
```

**Why `kubectl_manifest` instead of `kubernetes_manifest`:** The ArgoCD Application CRD is not registered until the Helm chart installs it. `kubernetes_manifest` validates CRD schema at plan time and would fail. `kubectl_manifest` (alekc/kubectl) defers validation to apply time.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Terraform -> K8s cluster | Helm/k8s providers write to cluster via kubeconfig; ArgoCD namespace + resources created |
| Bridge Secret annotations | Contain OCIDs (not auth material); readable by any pod in argocd namespace |
| Bootstrap Application | Points to external git repo (gitops-setup); controls what ArgoCD syncs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06 | Tampering | GitOps repo URL in Bridge Secret | mitigate | `gitops_repo_url` is a Terraform variable with default pointing to AssessForge org; changes require Terraform apply with explicit variable override |
| T-01-07 | Information Disclosure | Bridge Secret OCID values | accept | OCIDs are not auth material; Bridge Secret is cluster-internal only; no sensitive credentials stored in labels/annotations |
| T-01-08 | Elevation of Privilege | ArgoCD bootstrap Application with selfHeal=true | mitigate | `prune = false` prevents auto-deletion of resources; Application scoped to `bootstrap/control-plane` path only |
| T-01-09 | Spoofing | ArgoCD admin access during bootstrap | accept | Admin enabled temporarily; ClusterIP service means no external access; Phase 3 disables admin via GitOps |
| T-01-10 | Denial of Service | helm_release ignore_changes preventing fixes | accept | Intentional tradeoff for GitOps self-management; initial values verified correct; post-bootstrap changes via GitOps repo only |
</threat_model>
