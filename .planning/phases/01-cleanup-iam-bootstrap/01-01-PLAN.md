**1. Fix IAM Dynamic Group (per D-01, D-02 — IAM-01, IAM-02):**

In `terraform/infra/modules/oci-iam/main.tf`:
- Rename the resource label from `oci_identity_dynamic_group.workload_identity` to `oci_identity_dynamic_group.instance_principal`
- Change `name` from `"assessforge-workload-identity"` to `"assessforge-instance-principal"`
- Change `description` to `"Dynamic group para worker nodes OKE via Instance Principal (BASIC tier)"`

  `"ALL {resource.type = 'workload', resource.compartment.id = '${var.compartment_ocid}'}"`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator workstation -> OCI IAM | Terraform apply modifies IAM policies; operator must have IAM permissions |
| OKE worker nodes -> OCI Vault | Instance Principal auth via Dynamic Group; matching rule controls scope |
| Kubeconfig file -> K8s API | `~/.kube/config-assessforge` grants cluster access; must never be committed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | Dynamic Group matching rule | mitigate | Scope by `instance.compartment.id` — only instances in the specific compartment join the group, not all tenancy instances |
| T-01-02 | Spoofing | Kubeconfig file | mitigate | `~/.kube/config-assessforge` listed in CLAUDE.md "Files That Must Never Be Committed"; `.gitignore` convention; Helm/k8s providers reference via `pathexpand()` not inline |
| T-01-03 | Information Disclosure | ArgoCD admin password | accept | Admin enabled during bootstrap window only; Phase 3 disables admin via GitOps self-management; no external access yet (ClusterIP service) |
| T-01-04 | Denial of Service | Dynamic Group rename IAM gap | accept | `create_before_destroy` minimizes window; no live ESO workloads depend on it yet |
| T-01-05 | Tampering | Terraform state file | mitigate | Remote state in OCI Object Storage with S3 backend; `prevent_destroy` on critical resources (OKE, VCN, Vault, master key) |
</threat_model>
