<objective>
Fix the OKE worker node registration timeout by adding the missing Network Security Group rules between the `api_endpoint` NSG and the `workers` NSG inside `terraform/infra/modules/oci-network/main.tf`.

Purpose: The current `api_endpoint` NSG only allows bastion -> API (port 6443) ingress. It is missing every rule required for worker <-> control-plane communication per OCI OKE docs (https://docs.oracle.com/en-us/iaas/Content/ContEng/Concepts/contengnetworkconfig.htm). Without these rules, the `oci_containerengine_node_pool.main` resource fails with `2 nodes(s) register timeout` because the kubelet on each worker cannot reach the private API endpoint (TCP 6443 + TCP 12250), and the control plane cannot reach back to the kubelet (TCP 10250).

Output: Eight new `oci_core_network_security_group_security_rule` resources added to `terraform/infra/modules/oci-network/main.tf`. No variable/output/module-interface changes. No k8s layer changes.
</objective>

Existing conventions to mirror:
- protocol "6"  = TCP
- protocol "1"  = ICMP
- protocol "all" = all protocols
- source_type / destination_type = "NETWORK_SECURITY_GROUP" when using another NSG as peer

- snake_case resource labels

Rule shape reference (from `workers_ingress_from_lb`):
```hcl
resource "oci_core_network_security_group_security_rule" "<label>" {
  network_security_group_id = oci_core_network_security_group.<target>.id
  direction                 = "INGRESS" | "EGRESS"
  protocol                  = "6" | "1" | "all"
  source                    = <peer NSG id or CIDR>        # only for INGRESS
  source_type               = "NETWORK_SECURITY_GROUP" | "CIDR_BLOCK"
  destination               = <peer NSG id or CIDR>        # only for EGRESS
  destination_type          = "NETWORK_SECURITY_GROUP" | "CIDR_BLOCK"

  # For TCP with ports:
  tcp_options {
    destination_port_range { min = X, max = Y }
  }

  # For ICMP type 3 code 4 (path MTU discovery):
  icmp_options {
    type = 3
    code = 4
  }
}
```

ICMP 3/4 rules have NO `tcp_options`; they have `icmp_options { type = 3; code = 4 }` and `protocol = "1"`.

```hcl
# --- Regras api_endpoint <-> workers (exigidas pela OKE para registro dos nodes) ---
# Referencia: https://docs.oracle.com/en-us/iaas/Content/ContEng/Concepts/contengnetworkconfig.htm
```

Then add the following EIGHT rules, in this exact order and with these exact labels:

1. **api_endpoint_ingress_workers_kubeapi** (INGRESS on api_endpoint NSG)
   - Comment: `# Workers precisam acessar o Kubernetes API (TCP 6443) para registrar kubelet e executar chamadas de cluster`
   - protocol `"6"`, source = `oci_core_network_security_group.workers.id`, source_type `"NETWORK_SECURITY_GROUP"`
   - tcp_options.destination_port_range: min=6443, max=6443

2. **api_endpoint_ingress_workers_okeport** (INGRESS on api_endpoint NSG) -- CRITICAL para destravar o timeout
   - Comment: `# Canal dedicado OKE worker-to-control-plane (TCP 12250) -- sem esta regra os nodes nunca concluem o registro`
   - protocol `"6"`, source = workers NSG, source_type NSG
   - tcp_options.destination_port_range: min=12250, max=12250

3. **api_endpoint_ingress_workers_pmtu** (INGRESS on api_endpoint NSG)
   - Comment: `# Path MTU Discovery dos workers ate o API endpoint (ICMP type 3 code 4) -- evita blackhole de pacotes grandes`
   - protocol `"1"`, source = workers NSG, source_type NSG
   - icmp_options: type=3, code=4

4. **api_endpoint_egress_workers_kubelet** (EGRESS on api_endpoint NSG)
   - Comment: `# Control plane precisa alcancar a kubelet dos workers (TCP 10250) para exec/logs/metrics`
   - protocol `"6"`, destination = workers NSG, destination_type NSG
   - tcp_options.destination_port_range: min=10250, max=10250

5. **api_endpoint_egress_workers_pmtu** (EGRESS on api_endpoint NSG)
   - Comment: `# Path MTU Discovery do control plane para os workers (ICMP type 3 code 4)`
   - protocol `"1"`, destination = workers NSG, destination_type NSG
   - icmp_options: type=3, code=4

6. **api_endpoint_egress_oci_services** (EGRESS on api_endpoint NSG)
   - Comment: `# Control plane alcanca o servico OKE e demais OCI Services via Service Gateway (HTTPS 443)`
   - protocol `"6"`, destination = `data.oci_core_services.all.services[0].cidr_block`, destination_type `"SERVICE_CIDR_BLOCK"`
   - tcp_options.destination_port_range: min=443, max=443

7. **workers_ingress_from_api_endpoint_kubelet** (INGRESS on workers NSG)
   - Comment: `# Control plane -> kubelet nos workers (TCP 10250) -- obrigatorio para exec, logs e port-forward`
   - protocol `"6"`, source = `oci_core_network_security_group.api_endpoint.id`, source_type NSG
   - tcp_options.destination_port_range: min=10250, max=10250

8. **workers_ingress_from_api_endpoint_pmtu** (INGRESS on workers NSG)
   - Comment: `# Path MTU Discovery do control plane para os workers (ICMP type 3 code 4)`
   - protocol `"1"`, source = api_endpoint NSG, source_type NSG
   - icmp_options: type=3, code=4

Rationale notes (DO NOT put these in the file):
- Rule #6 uses `SERVICE_CIDR_BLOCK` destination_type (not `CIDR_BLOCK`) because the Service Gateway routes traffic via the Services OCID. See the existing `oci_core_route_table.private` rule that already uses this same `data.oci_core_services.all.services[0].cidr_block` with SERVICE_CIDR_BLOCK.
- `workers_egress_all` already permits all egress, so workers -> api_endpoint traffic (kubeapi 6443, okeport 12250, PMTU) is already allowed on the egress side -- no new workers-egress rules needed.
- Rules #1 and #2 are stateful on OCI NSGs, so reply traffic is implicit.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| workers NSG <-> api_endpoint NSG | East-west traffic between data plane nodes and the OKE-managed private control plane |
| api_endpoint NSG -> OCI Services (Service Gateway) | Control plane egress to OCI-managed OKE service endpoint over the private Oracle backbone |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-va5-01 | Information Disclosure | api_endpoint NSG ingress rules | mitigate | Source is `NETWORK_SECURITY_GROUP` (workers NSG), not `0.0.0.0/0`; only nodes attached to the workers NSG can reach 6443/12250 |
| T-va5-02 | Spoofing | workers NSG ingress from api_endpoint | mitigate | Source is api_endpoint NSG (OCI-managed), not a CIDR -- only the private OKE control plane VNIC can reach kubelet on 10250 |
| T-va5-03 | Elevation of Privilege | Overly broad kubelet ingress on workers | mitigate | Limit to TCP 10250 only (not all TCP) from api_endpoint NSG; do not use `protocol = "all"` |
| T-va5-04 | Denial of Service | Missing PMTU ICMP rules causing silent packet loss | mitigate | Rules #3, #5, #8 explicitly allow ICMP type 3 code 4 in both directions to avoid PMTU blackhole |
| T-va5-05 | Tampering | Rule drift via console edits | accept | Freeform tag `project = argocd-assessforge` + Terraform state ownership detects drift on next `terraform plan` |
