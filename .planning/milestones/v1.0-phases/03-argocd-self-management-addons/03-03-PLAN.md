Three files to create/update:

**1. Create `addons/cert-manager/manifests/cluster-issuer.yaml`** (per D-07, D-09, D-11, CERT-02):
```yaml
# ClusterIssuer — Let's Encrypt producao com solver HTTP-01 via Gateway API
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: admin@assessforge.com
    privateKeySecretRef:
      name: letsencrypt-prod-key
    solvers:
    - http01:
        gatewayHTTPRoute:
          parentRefs:
          - name: assessforge-gateway
            namespace: envoy-gateway-system
            kind: Gateway
```
The `gatewayHTTPRoute` solver tells cert-manager to create temporary HTTPRoutes on the `assessforge-gateway` (created in Plan 02) for ACME challenge verification. The Gateway HTTP listener (port 80) with `allowedRoutes.namespaces.from: All` allows cert-manager to attach these solver routes.

**2. Update `addons/cert-manager/application.yaml`** — add third source for manifests directory and add `SkipDryRunOnMissingResource=true` to syncOptions (per D-11, research Pitfall 1):

Add this as the third entry in the `sources` list:
```yaml
  - repoURL: 'https://github.com/AssessForge/gitops-setup'
    targetRevision: 'main'
    path: 'bootstrap/control-plane/addons/cert-manager/manifests'
```

Add `SkipDryRunOnMissingResource=true` to the existing `syncOptions` list. This prevents the CRD race condition where ArgoCD tries to dry-run the ClusterIssuer before the cert-manager CRDs are installed. Final syncOptions:
```yaml
    syncOptions:
    - CreateNamespace=true
    - ServerSideApply=true
    - SkipDryRunOnMissingResource=true
```

Keep all other content unchanged (metadata, existing sources, destination, syncPolicy automated settings).

**3. Replace stub in `environments/default/addons/cert-manager/values.yaml`** (per CERT-01):
```yaml
# Valores cert-manager — instala CRDs e habilita suporte a Gateway API
crds:
  enabled: true
  keep: true

config:
  apiVersion: controller.config.cert-manager.io/v1alpha1
  kind: ControllerConfiguration
  enableGatewayAPI: true
```
Do NOT use the deprecated `installCRDs: true`. Use `crds.enabled: true` (cert-manager v1.17+). `config.enableGatewayAPI: true` replaces the old `--feature-gates=ExperimentalGatewayAPISupport=true` flag (removed in v1.15).

**CERT-03 coverage note:** The Certificate for argocd.assessforge.com is created automatically by cert-manager when it detects the `cert-manager.io/cluster-issuer: letsencrypt-prod` annotation on the Gateway resource (created in Plan 02, gateway.yaml). cert-manager reads the HTTPS listener's hostname (`argocd.assessforge.com`) and `certificateRefs` Secret name (`argocd-tls`), then creates a Certificate resource in the `envoy-gateway-system` namespace that populates the `argocd-tls` Secret. No separate Certificate manifest is needed — this is the annotation-driven approach per D-08.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ACME server -> cert-manager | Let's Encrypt validates domain ownership via HTTP-01 challenge |
| cert-manager -> Gateway | Temporary HTTPRoutes created for ACME challenge response |
| kubelet -> metrics-server | kubelet-insecure-tls trusts self-signed kubelet certs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Spoofing | ACME domain validation | mitigate | HTTP-01 challenge proves domain control; only the domain owner can respond to ACME challenges via the Gateway |
| T-03-12 | Information Disclosure | TLS private key | mitigate | Private key stored as K8s Secret (`letsencrypt-prod-key`); cert-manager manages lifecycle; never in git |
| T-03-13 | Tampering | kubelet-insecure-tls | accept | Required for OKE private cluster; kubelet is on private subnet (10.0.2.0/24) behind NSG; risk is contained to cluster-internal traffic |
| T-03-14 | Denial of Service | cert-manager CRD race | mitigate | `SkipDryRunOnMissingResource=true` + `selfHeal: true` ensures convergence after CRDs install |
</threat_model>
